Notifications

Introduction

Notification is a mechanism to implement in order to automatically synchronize payments to your merchant site. Paymetrust will call this link after each update to notify you of status changes during a transaction. At the end of a payment, Paymetrust systematically calls the notification URL for the service concerned. The purpose of this call is to inform the merchant of the status of their transaction. The merchant can then validate their order if the payment is verified and accepted.

Transaction Notification

When the transaction (payment or transfer) has a final status (SUCCESS or FAILED)! Paymetrust will send a payment notification to your notify_url with the following parameters:

  • notify_token: this is the same as the one present in the web payment response, make sure this token matches the token sent in the POST notification. This way, you will validate the authenticity of the notification.

  • merchant_transaction_id: represents the unique identifier of your order sent in the request

  • transaction_id: represents the unique identifier of the order for this operation.

  • The request body is in JSON format as follows:

{ "notify_token": "f92069a6b56c4bbd93ce1824ca005b8d", "merchant_transaction_id": "63e0fe766f390", "transaction_id": "50901a80c84b4edcb4f50ae864bfa7c5", "user": { "name": "Findel Fofana", "email": "overlord@gmail.com", "phone_number": "+2250101561365" } }

With the notification, you don't have to do the manual check, you can change the transaction status with the notification response.
